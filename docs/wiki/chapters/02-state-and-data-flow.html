The Source of Truth for Autonomous Operations

Overview

In a graph-native architecture, state is the only way nodes communicate. The AgentState acts as a "Shared Memory" that persists across machine reboots, ensuring that the agent never loses its place in a multi-step manufacturing sequence.

The AgentState Object

Defined as a TypedDict, the state tracks every aspect of the agent's perception and memory.

Field Type Responsibility
v2_state PlannerOwnedV2State The planner-owned agenda, requirement ledger, evidence ledger, and response contract for the current run.
tool_outputs Annotated[list, add] An append-only trace of all API results.
staged_writes replaceable_list Buffer for 2PC pending mutations.
decisions List[PlannerDecision] The planner's reasoning history.

Custom Reducer Patterns

We use LangGraph reducers to control how node outputs merge into the global state. This is critical for preventing "Ghost Data" from previous failed loops.

Replaceable List Reducer

Unlike the default append reducer, this allows the system to clear or replace a list. We use this for staged_writes so that if a dry-run fails, the planner can replace the entire bundle with a new, corrected one.

def replaceable_list_reducer(old: list, new: list) -> list:
    return new if new is not None else old

Checkpoint Persistence

Every node transition triggers a State Snapshot. The snapshot is serialized using pickle or json and stored in the MySQL checkpoints table. This architecture ensures:

  • Survivability: Sessions can be resumed on different physical servers.
  • Auditability: Every "thought" and "action" is permanently recorded.
  • Replayability: We can re-run a specific session for debugging by hydrating from a specific checkpoint ID.
