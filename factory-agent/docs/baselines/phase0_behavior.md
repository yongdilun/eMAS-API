## Critical API Behavior Notes

- Sessions and messages are covered by existing endpoint tests for create, read, update, delete, snapshot, cancel, and message append flows.
- Plan creation is covered for persisted plan rows, planner adapter fallback, draft validation, graph-native approval, clarification, validation rejection, and tool scope behavior.
- Approval behavior is covered by endpoint and atomicity tests for approve, reject, stale approval handling, pending approval reads, and snapshot projection.
- SSE/activity/semantic endpoints are present in the OpenAPI baseline. Authentication hardening remains a Phase 2 issue per FA-001.
- DLQ behavior is covered for push, replay, replay-request, dismiss, and admin reads. Authentication hardening remains a Phase 2 issue per FA-001.
- Metrics endpoint currently exposes Prometheus text format and is covered by `test_metrics_endpoint_exposes_prometheus_format`. Exposure policy remains a Phase 2 issue per FA-001.
- Startup schema compatibility DDL remains present in `main.py` and is deferred to Phase 5 per FA-004.

## Baseline Friction Observed

- Capturing OpenAPI through `TestClient(main.app)` triggered a shutdown `CancelledError` from the lifespan path. The saved snapshot was generated by building an equivalent FastAPI app with `build_router(...)` directly, avoiding startup side effects.
- Docker commands require access outside the default sandbox because Docker needs the user Docker config and daemon pipe.
